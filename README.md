# 🎣 บ่อตาน้ำ ฟิชชิ่งพาร์ค — ระบบสมาชิกบ่อตกปลา

ระบบสมาชิกบ่อตกปลาครบวงจร สร้างด้วย **Next.js 15 (App Router)** ทั้ง Frontend และ Backend
ฐานข้อมูล SQLite ผ่าน `node:sqlite` (มากับ Node.js — **ไม่ต้องติดตั้งฐานข้อมูลแยก**)

## ฟีเจอร์

| ฟีเจอร์ | รายละเอียด |
|---|---|
| 🎫 QR เข้าบ่อ | QR ส่วนตัวแบบ Dynamic หมุนใหม่ทุก 90 วินาที (HMAC) กันแคปหน้าจอส่งต่อ + รหัสสำรอง 6 หลัก |
| 💧 Wallet | เติมเงินแบบแพ็กเกจมีโบนัส พนักงานอนุมัติ ยอดเข้าอัตโนมัติ ดูประวัติได้ |
| ✅ Check-in | สแกนแล้วตัด 100 บาท/วัน อัตโนมัติ กันตัดซ้ำวันเดียวกัน กันยอดไม่พอ +10 แต้ม/ครั้ง |
| 📸 ส่งรูปปลา | ถ่ายรูป + ชนิด + น้ำหนัก พนักงานยืนยันก่อนขึ้นกระดาน (+5 แต้ม) มีอัลบั้มส่วนตัว |
| 🏆 Ranking | 4 กระดานรายเดือน: ปลาใหญ่สุด / จำนวนตัว / น้ำหนักรวม / ขาประจำ |
| 🧑‍💼 หน้าพนักงาน | แดชบอร์ดสถิติรายวัน สแกนรับเข้าบ่อ อนุมัติเติมเงิน ยืนยันปลา |

## วิธีรัน

ต้องใช้ **Node.js 22 ขึ้นไป** (เพราะใช้ `node:sqlite`)

```bash
npm install
npm run dev        # โหมดพัฒนา → http://localhost:3000
# หรือใช้งานจริง
npm run build && npm start
```

เปิดครั้งแรกระบบจะสร้างฐานข้อมูล `pond.db` พร้อมข้อมูลตัวอย่างให้อัตโนมัติ

### บัญชีทดลอง (login ด้วยเบอร์โทร)

- สมาชิก: `0811111111`, `0822222222`, `0833333333`
- พนักงาน/แอดมิน: `0800000000`
- เบอร์อื่น = สมัครสมาชิกใหม่ (กรอกชื่อเพิ่ม)

## Flow การเช็คอินหน้าบ่อ

1. สมาชิกเปิดหน้า **เข้าบ่อ** → เห็น QR ส่วนตัว (หมดอายุทุก 90 วิ รีเฟรชเอง)
2. พนักงาน (login บนมือถือ/แท็บเล็ตของร้าน) เปิดกล้องสแกน QR
   → QR พาเข้า `/admin/scan?d=...` และตัดเงินอัตโนมัติ
3. หรือพิมพ์ **รหัสสมาชิก + รหัส 6 หลัก** ใต้ QR แทน
4. ระบบตัด 100 บาท บันทึกเช็คอิน ให้แต้ม และแจ้งผลทั้งสองฝั่ง

> QR encode เป็น `userId.HMAC(userId, timeWindow)` — ปลอมไม่ได้ หมดอายุเอง
> และหน้า QR ของสมาชิกจะเด้งเป็น "เช็คอินแล้ว ✅" ภายใน 5 วินาทีหลังสแกน

## โครงสร้างสำคัญ

```
src/lib/db.ts        ฐานข้อมูล + schema + seed (node:sqlite)
src/lib/auth.ts      session cookie (HMAC-signed)
src/lib/token.ts     Dynamic QR code หมุนทุก 90 วิ
src/lib/wallet.ts    ค่าเข้าบ่อ + แพ็กเกจเติมเงิน (แก้ราคาที่นี่)
src/app/api/…        Backend ทั้งหมด (REST)
src/app/…            หน้าจอสมาชิก + พนักงาน (mobile-first)
```

## ปรับแต่งที่พบบ่อย

- **ค่าเข้าบ่อ / แพ็กเกจเติมเงิน** → `src/lib/wallet.ts` (และ `PACKAGES` ใน `src/app/wallet/TopUpForm.tsx`)
- **ชนิดปลาในฟอร์ม** → `SPECIES` ใน `src/app/catch/CatchForm.tsx`
- **ชื่อบ่อ / ประกาศ / เวลาเปิดปิด** → `src/app/page.tsx`, `src/app/layout.tsx`
- **Secret ใช้งานจริง** → ตั้ง `SESSION_SECRET` ใน `.env.local` เป็นค่าสุ่มยาวๆ

## ต่อยอดเข้ากับ LINE (LIFF)

โปรเจกต์นี้เป็นเว็บแอป mobile-first ที่พร้อมฝังใน LINE:
1. สร้าง LINE Login channel + LIFF app ชี้มาที่โดเมนของระบบนี้
2. แทนหน้า `/login` ด้วย `liff.init()` + `liff.getProfile()` แล้วยิงเข้า `/api/auth/login` โดยใช้ `userId` ของ LINE แทนเบอร์โทร
3. ใส่ URL ของ LIFF ใน Rich Menu ของ LINE OA (เข้าบ่อ / กระเป๋า / ส่งปลา / อันดับ)
4. ใช้ Messaging API push แจ้งเตือน (เช็คอินสำเร็จ, ยอดใกล้หมด, ประกาศลงปลา)

## หมายเหตุการใช้งานจริง

- ระบบนี้เก็บไฟล์รูปที่ `public/uploads` และฐานข้อมูลเป็นไฟล์ → เหมาะกับการ deploy บน VPS/เครื่องเดียว (เช่น PM2 + Nginx) ไม่เหมาะกับ serverless
- ควรเพิ่ม OTP ยืนยันเบอร์โทรก่อนเปิดใช้จริง
